Theoretical Differences of Sorting Algorithms

Algorithm Type Best Case Time Complexity Average/Worst Case Time Complexity Space Complexity Stability Description
Insertion Sort Comparison-based O(n) O(n²) O(1) Stable Builds a sorted array incrementally by taking one element at a time.
Bubble Sort Comparison-based O(n) O(n²) O(1) Stable Repeatedly traverses the array, comparing and swapping adjacent elements.
Selection Sort Comparison-based O(n²) O(n²) O(1) Not Stable Divides the array into sorted and unsorted parts, selecting the smallest element.
Merge Sort Comparison-based O(n log n) O(n log n) O(n) Stable Divides the array into halves and merges them in sorted order.
Bucket Sort Non-comparison-based O(n + k) O(n + k) O(n + k) Stable Distributes elements into buckets, sorting each individually.
Quick Sort Comparison-based O(n log n) O(n²) O(log n) Not Stable Picks a pivot and partitions the array into elements less than or greater than the pivot.
Heap Sort Comparison-based O(n log n) O(n log n) O(1) Not Stable Builds a max heap and repeatedly extracts the maximum element.
Counting Sort Non-comparison-based O(n + k) O(n + k) O(k) Stable Counts occurrences of each value and determines positions in sorted output.
Radix Sort Non-comparison-based O(nk) O(nk) O(n + k) Stable Sorts numbers by processing individual digits from least to most significant.

Note: "n" refers to the number of elements, and "k" refers to the range of the input values.
